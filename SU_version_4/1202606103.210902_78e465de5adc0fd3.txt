------------ communication -----------------------

each job itself is 24 bytes,  becuase each zv is 24 bytes,           (3 nats, each 8 bytes!)


	and we are sending a set of jobs to the client worker thread.	


the number of jobs which we send in a packet,  is some parameter to the system,   called    jobs_per_packet

and so the number of bytes which we send over the network from the server to the client, should be: 



		1 + jobs_per_packet * 24  +  1



	first 1 is for the client the request

	then jobs_per_packet * 24 is the payload, which is the jobset the client will work on now

	then the final 1 is the client's ack of the jobset payload being received. 







so,

			0x8F        :    requests a new job

							        [ byte0 byte1 byte2 byte3  byte4 byte5 byte6 byte7 ]      (g2 nat)


						etc
